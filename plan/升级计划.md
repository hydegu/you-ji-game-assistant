# 游记助手（YouJiAssistant）全面升级计划

## 背景与目标

项目目前是约10个Java文件的基础脚手架（只有Milvus向量搜索，无MySQL、无认证、无LLM回答生成、无文件上传）。目标是将其升级为功能完整的游戏攻略问答助手：

- **知识库管理模块**：管理员上传PDF → 文本分块 → 向量化 → 存入Milvus
- **问答检索核心**：问题向量化 → Milvus Top-k召回 → 拼装Prompt → LLM流式回答
- **对话历史功能**：MySQL存储问答记录、收藏、最近3轮上下文注入
- **JWT权限控制**：管理员管理知识库，普通用户问答+查看历史

## 阶段三：知识库管理（管理员功能）

**文本分块算法**（实现于 `DocumentChunkingServiceImpl`）：

1. **段落优先分割**：按 `\n\n+` 切分，清理空白，过滤长度 < 20字符的噪声段落
2. **Token估算**：`text.length() / 2.5`（适用于中英混合文本）
3. **合并小段落**：累积多个短段落直到超过 MAX_TOKENS（400），作为一个Chunk输出
4. **滑动窗口兜底**（处理超长单段落）：
   - 按句子边界 `[。！？.!?]` 再次切分
   - 超过 MAX_TOKENS 时输出当前窗口，保留末尾 OVERLAP_TOKENS（50）的内容作为下个窗口的前缀
5. **Chunk元数据**：`{ gameId, docId, fileName, chunkIndex, chunkTotal }`（docId用于按文档删除向量）

参数配置（`application.yaml`）：
```yaml
app.chunking.max-tokens: 400
app.chunking.min-tokens: 80
app.chunking.overlap-tokens: 50
```

**需要新建的文件：**
- `entity/KnowledgeDoc.java` + mapper + XML
- `service/DocumentChunkingService.java`（接口）+ `impl/DocumentChunkingServiceImpl.java`（分块算法）
- `service/KnowledgeService.java`（接口）+ `impl/KnowledgeServiceImpl.java`：
  - 使用已有的 `spring-ai-pdf-document-reader` 解析PDF提取文本
  - 调用 `DocumentChunkingService` 分块
  - 封装为带元数据的 Spring AI `Document` 对象
  - 调用 `gameRagService.ingestDocument(gameId, docs)` 存入Milvus
  - 在MySQL中写入 `KnowledgeDoc` 记录
  - 删除时：按 `docId` 元数据过滤删除Milvus向量，同时软删除MySQL记录
- `controller/KnowledgeController.java`：
  - `POST /admin/knowledge/upload`（multipart：file + gameId）
  - `DELETE /admin/knowledge/{docId}`
  - `GET /admin/knowledge/{gameId}`

## 阶段四：问答核心（RAG + LLM + 流式输出）

**需要修改的已有文件：**
- `service/GameRagService.java` — 添加 `searchWithTopK(Long gameId, String query, int topK)` 重载
- `resources/prompts/system-qa.st` — 将硬编码示例替换为参数占位符（保留所有引用规则和输出格式规则）：
  ```
  {context}  = Milvus召回结果："[1] 来源：《文件名》\n内容：...\n\n[2] ..."
  {history}  = 最近3轮对话："用户: ...\n助手: ...\n\n" 或 "（暂无历史对话）"
  {question} = 用户当前问题
  ```

**需要新建的文件：**
- `entity/QARecord.java` + `mapper/QARecordMapper.java` + XML（自定义：`findRecentByUserAndGame(userId, gameId, limit)`）
- `dto/request/ChatRequest.java` — `{ gameId, question }`
- `service/ChatService.java` + `impl/ChatServiceImpl.java`，核心流程：
  1. 查询 `(userId, gameId)` 最近3条 QARecord（DESC LIMIT 3，翻转为时间正序）
  2. 格式化历史对话块
  3. 调用 `gameRagService.searchWithTopK(gameId, question, 5)` 组装上下文块
  4. 用 `system-qa.st` 模板构建 `Prompt`
  5. 通过 `chatClient.prompt(prompt).stream().content()` 获得 `Flux<String>`
  6. 在 `doOnComplete` 中将完整回答持久化到 `qa_records`
- `controller/ChatController.java`：
  - `POST /chat/ask` → `produces = text/event-stream`，返回 `Flux<String>`（SSE流式）
  - `GET /chat/games` → 列出所有游戏
